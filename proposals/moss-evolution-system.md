## 🎯 核心目标

让 MOSS 具备**自我发现、自我学习、自我优化**的能力，不需要老板提醒就能持续进化。

---

## 📐 设计原则

基于 WORK_PRINCIPLES.md 15 条准则：

| 原则 | 进化系统设计 |
|------|-------------|
| 原则一（法律边界） | 所有自动学习/执行操作必须在法律允许范围内 |
| 原则二（24h 开发） | 空闲时自动启动进化流程 |
| 原则三（创业思维） | 进化方向优先指向增收能力 |
| 原则四（质量第一） | 新技能必须经过测试才能启用 |
| 原则五（自我反思） | 每次会话后自动反思并记录 |
| 原则六（定时汇报） | 进化进展纳入 hourly report |
| 原则七（能力提升） | 检测到能力缺口自动触发学习 |
| 原则八（Subagent 优先） | 进化任务本身用 subagent 执行 |
| 原则十五（资源管理） | 控制 API 调用成本，优先免费资源 |

## 🧬 进化维度

### 1️⃣ Subagent 指挥官能力

**目标：** 让 MOSS 成为高效的 subagent 指挥官

| 能力 | 现状 | 目标 | 进化方法 |
|------|------|------|---------|
| 任务分解 | 手动分析 | 自动拆解 | 建立任务模式库 |
| 并行编排 | 基础并行 | 智能调度 | 依赖图 + 优先级队列 |
| 质量控制 | 事后检查 | 过程监控 | 设置检查点 |
| 冲突解决 | 人工介入 | 自动协调 | 冲突检测 + 协商机制 |

**实现路径：**
```
1. 建立任务模板库（proposals/task-patterns.md）
2. 实现 subagent 调度器（scripts/subagent-orchestrator.sh）
3. 添加质量检查点（scripts/quality-gate.sh）
4. 冲突解决规则（docs/conflict-resolution.md）
```

---

### 2️⃣ AI 合伙人能力

**目标：** 让 MOSS 具备商业洞察和主动提案能力

| 能力 | 现状 | 目标 | 进化方法 |
|------|------|------|---------|
| 商业洞察 | 被动响应 | 主动发现机会 | 市场扫描 + 趋势分析 |
| 主动提案 | 等待指令 | 定期输出方案 | 头脑风暴自动化 |
| 风险预警 | 事后报告 | 提前预警 | 风险检测规则 |
| 资源优化 | 人工评估 | 自动优化建议 | ROI 计算模型 |

**实现路径：**
```
1. 市场扫描脚本（scripts/market-scan.sh）
2. 自动提案生成器（scripts/proposal-generator.sh）
3. 风险检测规则（docs/risk-rules.md）
4. ROI 计算器（scripts/roi-calc.sh）
```

---

### 3️⃣ 执行能力

**目标：** 让 MOSS 具备自动化、工具链、知识沉淀能力

| 能力 | 现状 | 目标 | 进化方法 |
|------|------|------|---------|
| 自动化 | 基础脚本 | 全流程自动化 | 工作流引擎 |
| 工具链 | 分散工具 | 统一工具链 | 工具注册表 |
| 脚本封装 | 手动封装 | 自动封装 | 脚本生成器 |
| 知识沉淀 | 人工记录 | 自动沉淀 | 知识提取器 |

**实现路径：**
```
1. 工作流引擎（scripts/workflow-engine.sh）
2. 工具注册表（docs/tool-registry.md）
3. 脚本生成器（scripts/script-generator.sh）
4. 知识提取器（scripts/knowledge-extractor.sh）
```

- 新技能学习数量
- 自动化脚本新增数量
- 主动提案数量
- 能力缺口修复率

### 中期指标（每周）

- 任务完成效率提升
- subagent 利用率
- ROI 改善
- 知识沉淀质量

### 长期指标（每月）

- 技术竞争力提升

## ⚠️ 风险控制

### 自动进化边界

| 风险类型 | 控制措施 |
|----------|---------|
| 法律风险 | 所有学习内容先过法律边界检查 |
| 成本风险 | API 调用设置日限额 |
| 质量风险 | 新技能必须通过测试才能启用 |
| 方向风险 | 进化方向需符合老板战略 |
| 安全风险 | 代码变更需经过安全扫描 |

### 人工介入点

以下情况需要老板拍板：
- 涉及付费学习资源
- 重大方向调整
- 可能影响现有项目的变更
- 法律灰色地带
