# 数字孪生+AI 解说员 MVP 定义

> **版本：v1.0** | **日期：2026-03-07**  
> **开发周期：4 周** | **目标：验证核心价值假设**

---

## 一、MVP 目标

### 核心价值假设
> **为数字孪生场景添加 AI 解说员，可将访客/员工查询效率提升 3 倍，减少 50% 重复讲解工作**

### MVP 验证目标
| 指标 | 目标值 | 测量方式 |
|-----|-------|---------|
| **查询响应时间** | <3 秒 | 系统日志 |
| **回答准确率** | >85% | 用户反馈+ 人工抽检 |
| **用户满意度** | >4.0/5.0 | 试点客户调研 |
| **日活跃用户** | >20 人/天 | 使用统计 |
| **付费意向** | ≥2 家试点转付费 | 商务跟进 |

│                  前端（Web）                         │

│  │ (谷风现有)   │  │ (新增)      │  │ (新增)      │  │
│  └─────────────┘  └─────────────┘  └─────────────┘  │
└─────────────────────┬───────────────────────────────┘
                      │
                      │ HTTPS
                      │
┌─────────────────────▼───────────────────────────────┐

│  └─────────────────────┬───────────────────────┘    │
│                        │                             │
│    ┌───────────────────┼───────────────────┐        │
│    │                   │                   │        │
│    ▼                   ▼                   ▼        │
│ ┌────────┐      ┌────────────┐     ┌──────────┐    │
│ │ 谷风 API │      │ AI 服务     │     │ 知识库    │    │
│ │ 适配器  │      │ (Python)    │     │ (Milvus) │    │
│ └────────┘      └─────┬──────┘     └──────────┘    │
│                       │                              │
│                       ▼                              │

│              │ LLM (Qwen API) │                     │
│              └────────────────┘                     │
└─────────────────────────────────────────────────────┘

### 技术栈明细

| 层级 | 技术选型 | 理由 |
|-----|---------|------|
| **前端** | Vue3 + Element Plus | 与谷风现有技术栈一致 |
| **后端** | Python FastAPI | AI 生态丰富，开发效率高 |
| **LLM** | Qwen2.5-72B（阿里云 API） | 中文能力强，成本低 |
| **向量库** | Milvus Lite（嵌入式） | 免运维，MVP 够用 |
| **部署** | Docker + 谷风现有服务器 | 复用基础设施 |

## 六、成功标准

### MVP 验收标准
- [ ] P0 功能全部实现并通过测试
- [ ] 响应时间<3 秒（P95）
- [ ] 试点客户完成培训并开始使用
- [ ] 收集≥50 条真实用户反馈
- [ ] 至少 1 家试点表达付费意向

### 继续/终止决策点
| 条件 | 决策 |
|-----|------|
| 3 家试点全部满意 + 2 家付费意向 | **继续**：启动正式版本开发 |
| 2 家试点满意 + 1 家付费意向 | **调整**：优化后扩大试点 |
| <2 家试点满意 | **暂停**：重新评估需求/方案 |

## 七、风险与缓解

| 风险 | 缓解措施 |
|-----|---------|
| **LLM 回答不准确** | RAG 增强+ 设置置信度阈值+ 人工审核开关 |
| **集成难度大** | 第 1 周完成 API 调研，如有问题及时调整方案 |
| **试点客户配合度低** | 选择高意愿客户+ 磊哥亲自背书+ 专人对接 |
| **进度延期** | 每周站会同步进度，必要时砍 P1 功能保 P0 |
