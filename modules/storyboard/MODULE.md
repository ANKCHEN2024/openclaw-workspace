## 🎯 功能概述

分镜生成模块是 AI 短剧平台的核心模块，负责将剧本文本自动转换为可视化的分镜图像序列。

### 核心功能

1. **文生图生成** - 使用阿里通义万相 API 生成图像
2. **多镜头支持** - 8 种镜头角度（大远景到特写）
3. **人物一致性** - 保持人物在不同分镜中的视觉一致性
4. **场景一致性** - 保持场景在不同分镜中的视觉一致性
5. **批量生成** - 支持一次生成多个镜头的多张图像
6. **异步任务** - 支持异步生成和进度查询
7. **智能评分** - 自动计算一致性分数和质量评分

## 📊 数据结构

### 主要数据模型

1. **Storyboard** - 分镜
   - 包含场景描述、人物、动作、镜头角度
   - 关联多个生成的图像

2. **StoryboardImage** - 分镜图像
   - 图像 URL、缩略图 URL
   - 生成参数（提示词、种子）
   - 镜头角度、构图信息

3. **CharacterDescription** - 人物描述
   - 外貌特征、发型、服装
   - 表情、姿态

4. **ConsistencyScore** - 一致性分数
   - 按镜头角度分组
   - 平均、最低、最高分数

## 🔐 一致性控制方案

### 方案 1: 固定种子

- 为人物和场景生成固定种子
- 相同种子确保相同生成结果
- 优点：简单可靠
- 缺点：灵活性较低

### 方案 2: 参考图 + ControlNet

- 使用人物/场景参考图
- 通过参考强度控制一致性
- 优点：效果好
- 缺点：需要额外存储参考图

### 方案 3: 提示词工程

- 在提示词中固定关键特征
- 使用一致性关键词
- 优点：无需额外配置
- 缺点：依赖 API 理解能力

### 方案 4: CLIP 相似度

- 使用 CLIP 模型计算视觉相似度
- 自动筛选一致性高的图像
- 优点：客观量化
- 缺点：需要额外模型

## 📈 性能指标

### 生成时间

- 单张图像：~15-30 秒
- 单镜头（4 张）：~30-60 秒
- 完整分镜（3 镜头×4 张）：~90-180 秒

### 一致性阈值

- **优秀**: >90 分
- **良好**: 85-90 分
- **可接受**: 75-85 分
- **需重新生成**: <75 分

## 🚀 使用流程

```
1. 初始化模块
   ↓
2. 准备输入数据（场景、人物、动作）
   ↓
3. 调用 generate() 或 generateAsync()
   ↓
4. 等待生成完成（或轮询状态）
   ↓
5. 查看生成的图像
   ↓
6. 选择最佳图像
   ↓
7. （可选）重新生成不满意的镜头
```

## ⚠️ 注意事项

1. **API 限流**: 通义万相 API 有调用频率限制，建议使用异步模式
2. **存储成本**: 生成的图像较多，注意 MinIO 存储容量
3. **一致性平衡**: 过高的一致性可能限制创意，根据需求调整
4. **错误处理**: 网络波动可能导致生成失败，实现重试机制
