# 📖 故事分析模块设计文档

## 模块概述

**模块名称：** Story Analysis Module (故事分析模块)  
**模块编号：** Subagent-02  
**API 提供商：** 阿里通义千问 (DashScope)  
**输入：** 故事大纲/小说文本  
**输出：** 结构化故事元素（主题、冲突、节奏、情感曲线）

---

## 1. 功能定位

故事分析模块是 AI 短剧生成平台的**核心入口模块**，负责：

1. **文本解析** - 接收并预处理小说/故事文本
2. **元素提取** - 识别故事的核心元素（主题、人物、冲突等）
3. **结构分析** - 分析故事节奏和情感曲线
4. **分集建议** - 根据内容自动建议分集方案
5. **风格识别** - 识别中文网络文学风格特征

## 3. 核心功能

### 3.1 故事元素提取

| 元素 | 描述 | 输出格式 |
|------|------|----------|
| 主题 | 故事核心主题和副主题 | Array<string> |
| 类型 | 故事类型标签 | Array<string> |
| 基调 | 整体情感基调 | string |
| 冲突 | 主要冲突和次要冲突 | Array<Conflict> |
| 节奏 | 情节节奏分析 | Rhythm |
| 情感曲线 | 情感强度变化 | Array<EmotionPoint> |

### 3.2 中文网络文学风格支持

支持识别以下风格特征：

- **玄幻/奇幻** - 修真、魔法、异世界
- **都市/言情** - 现代都市、职场、爱情
- **悬疑/推理** - 犯罪、侦探、谜团
- **历史/穿越** - 古代、穿越、重生
- **科幻/末世** - 未来科技、末日生存
- **武侠/仙侠** - 江湖、修仙、武功

### 3.3 分集建议

根据故事内容自动建议：

- 分集数量
- 每集核心情节
- 每集情感高潮点
- 悬念设置建议

## 4. 数据流

```
用户输入 (小说文本)
    ↓
[文本预处理]
    - 去除无关字符
    - 统一编码 (UTF-8)
    - 分段 (按章节/场景)
    - 长度控制 (<50KB)
    ↓
[调用通义千问 API]
    - 使用专用提示词模板
    - 设置 temperature=0.7
    - 设置 max_tokens=4096
    ↓
[解析 API 响应]
    - JSON 解析
    - 数据验证
    - 格式标准化
    ↓
[输出结构化数据]
    - 返回给调用方
    - 缓存到 Redis
    - 记录分析日志
```

## 6. 错误处理

### 6.1 错误类型

| 错误码 | 错误类型 | 处理策略 |
|--------|----------|----------|
| E001 | 文本过长 | 自动分段处理 |
| E002 | API 调用失败 | 重试 (3 次) + 降级 |
| E003 | JSON 解析失败 | 使用备用解析器 |
| E004 | 数据验证失败 | 返回部分结果 + 警告 |
| E005 | 超时 | 异步处理 + 回调通知 |

### 6.2 降级策略

当通义千问 API 不可用时：

1. 尝试切换到百度文心一言 API
2. 使用本地规则引擎进行基础分析
3. 返回缓存的历史分析结果 (如可用)
