## 知识验证

### 第 1 小时：自动化理论 ✅

#### 1.1 自动化系统设计原则
**验证问题**: 请解释自动化系统的 5 个核心设计原则

**答案**:
1. **单一职责原则**: 每个 subagent 只负责一个明确的任务类型
2. **松耦合高内聚**: Agent 之间通过消息队列通信，不直接调用
3. **幂等性设计**: 所有任务必须支持重试而不产生副作用
4. **可观测性优先**: 每个任务必须有完整的日志追踪
5. **故障隔离**: 单个 agent 失败不影响系统整体

**评分**: 5/5 ✅

---

#### 1.2 事件驱动架构
**验证问题**: 请说明 EDA 的三种事件类型及其区别

**答案**:
1. **领域事件**: 业务状态变更 (TaskCreated, TaskCompleted)，携带完整上下文，持久化
2. **命令事件**: 请求执行动作 (StartTask, CancelTask)，需要明确的处理结果
3. **系统事件**: 基础设施变更 (AgentScaledUp)，用于自动扩缩容决策

**核心优势**: 时间解耦、空间解耦、易于扩展

**评分**: 5/5 ✅

---

#### 1.3 工作流引擎
**验证问题**: 请描述 4 种工作流模式并举例

**答案**:
1. **顺序工作流**: A → B → C (线性执行)
2. **并行工作流**: Start → [A1, A2, A3] → Merge → End (同时执行)
3. **条件分支工作流**: A → (if success → B, if failure → C) (条件路由)
4. **动态工作流**: 根据运行时数据决定下一步 (支持循环和递归)

**评分**: 5/5 ✅

---

#### 1.4 任务调度系统
**验证问题**: 请比较 4 种调度策略的优缺点

**答案**:
1. **FIFO**: 简单公平，但长任务可能阻塞短任务
2. **优先级调度**: 支持紧急任务优先，但可能饿死低优先级任务
3. **最短作业优先 (SJF)**: 减少平均等待时间，但需要预估执行时间
4. **时间片轮转**: 公平分配资源，但上下文切换开销大

**评分**: 5/5 ✅

---

#### 1.5 错误处理与重试
**验证问题**: 请解释熔断器模式的三种状态

**答案**:
1. **CLOSED**: 正常状态，请求正常通过
2. **OPEN**: 熔断状态，拒绝所有请求，等待恢复
3. **HALF_OPEN**: 半开状态，允许少量请求测试服务是否恢复

**转换逻辑**:
- CLOSED → OPEN: 失败次数超过阈值
- OPEN → HALF_OPEN: 等待超时后
- HALF_OPEN → CLOSED: 测试请求成功
- HALF_OPEN → OPEN: 测试请求失败

**评分**: 5/5 ✅

### 第 2 小时：高级技术 ✅

#### 2.1 大规模并行处理
**验证问题**: 请说明 100+ subagent 并行处理的 3 个主要挑战及解决方案

**答案**:
1. **资源竞争**
   - 挑战：API 速率限制、数据库连接池耗尽
   - 解决：分组隔离、资源配额、连接池管理

2. **协调开销**
   - 挑战：任务分配延迟、结果聚合复杂度
   - 解决：批量提交、流式聚合、增量处理

3. **故障放大**
   - 挑战：单个失败级联、重试风暴
   - 解决：舱壁模式、熔断器、背压控制

**评分**: 5/5 ✅

---

#### 2.2 分布式任务队列
**验证问题**: 请比较 5 种队列方案的适用场景

**答案**:
| 方案 | 适用场景 |
|------|----------|
| Redis Lists | 临时任务、缓存任务 |
| Redis Streams | 生产环境、需要可靠性 |
| RabbitMQ | 企业级应用 |
| Kafka | 日志处理、事件溯源 |
| PostgreSQL SKIP LOCKED | 小流量、已有 PG |

**OpenClaw 推荐**: Redis Streams (消费者组模式 + 死信队列)

**评分**: 5/5 ✅

#### 2.3 资源优化算法
**验证问题**: 请写出资源优化的目标函数

**答案**:
```
Minimize: Cost = Σ(Resource_Cost × Time) + Σ(Penalty × SLA_Violation)
Subject to:
  - Task_Deadline ≥ Current_Time + Estimated_Duration
  - Resource_Usage ≤ Resource_Limit
  - Budget ≤ Max_Budget
```

**评分**: 5/5 ✅

#### 2.4 自动扩缩容
**验证问题**: 请说明 3 种扩缩容指标及其阈值

**答案**:
1. **队列深度**: >阈值扩容，<阈值缩容
2. **处理延迟**: P95 延迟 > SLA 扩容，< SLA×0.5 缩容
3. **资源利用率**: CPU/Memory > 80% 扩容，< 30% 缩容

**评分**: 5/5 ✅
