## 二、各 Bot 详细配置

### 2.1 MOSS-CEO 助理

#### 基本信息
| 配置项 | 值 | 备注 |
|--------|-----|------|
| 应用名称 | `MOSS-CEO 助理` | 对外显示名称 |
| 应用描述 | `磊哥的 AI 总助理，负责决策支持、项目协调与资源分配` | 应用市场可见 |
| 应用图标 | 建议：深蓝色机器人 + 领带/公文包元素 | 1080x1080 PNG |
| 应用类型 | 企业内部应用 | 不对外发布 |
| 可见范围 | 管理层成员 + 磊哥 | 按需配置 |

#### 权限配置

**必要权限**（必须开通）：
| 权限名称 | 权限码 | 申请理由 |
|----------|--------|----------|
| 机器人 | `robot` | 以机器人身份在群内发送消息 |
| 发送消息 | `im:message` | 向用户和群组发送通知和回复 |
| 读取消息 | `im:chat` | 读取群消息以理解上下文并响应 |
| 获取用户信息 | `contact:user.base` | 识别@提及的用户 |
| 获取群组信息 | `contact:group.base` | 识别消息来源群组 |
| 文档读取与写入 | `drive:file` | 管理战略文档、会议纪要 |
| 多维表格读取与写入 | `drive:table` | 访问 OKR 表、项目汇总表 |
| 日历读取与写入 | `calendar:calendar` | 管理磊哥日程、安排会议 |

**可选权限**（按需开通）：
| 权限名称 | 权限码 | 使用场景 |
|----------|--------|----------|
| 会议管理 | `meeting:meeting` | 创建和管理视频会议 |
| 云空间管理 | `drive:folder` | 访问公司云盘共享文件 |
| 审批管理 | `approval:approval` | 查看和发起审批流程 |
| 任务管理 | `task:task` | 创建和分配任务 |

#### 发布流程
1. 完成权限配置后，点击「版本管理」
2. 填写版本说明：`v1.0 - 初始版本，CEO 助理功能`
3. 点击「提交审核」（企业内部应用通常自动通过）
4. 审核通过后，点击「发布」
5. 在「可见范围」中添加管理层成员

## 三、凭证存储规范

### 3.1 1Password 命名规范

每个 Bot 在 1Password 中创建一个独立的「密码」项目，命名格式：

```
Feishu MOSS-{角色}
```

**示例**：
- `Feishu MOSS-CEO`
- `Feishu MOSS-PM`
- `Feishu MOSS-Finance`
- `Feishu MOSS-HR`
- `Feishu MOSS-Tech`

### 3.2 字段设计

每个密码项包含以下字段：

| 字段名 | 字段类型 | 示例值 | 来源 |
|--------|----------|--------|------|
| `app_id` | 文本 | `cli_a1b2c3d4e5f6g7h8` | 飞书应用基本信息 |
| `password` (app_secret) | 密码 | `XXXXXXXXXXXXXXXX` | 飞书应用凭证 |
| `verify_token` | 文本 | `your_verify_token` | 飞书应用凭证 |
| `encrypt_key` | 文本 | `your_encrypt_key` | 飞书应用凭证 |

**1Password 字段截图指引**：
1. 点击「添加字段」
2. 选择「文本字段」
3. 输入字段名（如 `app_id`）
4. 填写对应值

### 3.3 1Password 路径规范

OpenClaw 配置中使用的路径格式：

```
op://Personal/Feishu MOSS-CEO/app_id
op://Personal/Feishu MOSS-CEO/password
op://Personal/Feishu MOSS-CEO/verify_token
op://Personal/Feishu MOSS-CEO/encrypt_key
```

**说明**：
- `Personal`：1Password 保险库名称（根据实际情况调整）
- `Feishu MOSS-CEO`：密码项名称
- 最后一段：字段名

### 3.4 访问权限控制

**谁能访问**：
- ✅ 磊哥（管理员）
- ✅ 系统运维人员（如需）
- ❌ 其他人员（禁止）

**1Password 共享设置**：
1. 右键点击密码项 →「共享」
2. 仅添加必要的协作者
3. 设置为「可查看」或「可编辑」（建议可查看）

## 五、常见问题

### Q1: 权限申请被驳回怎么办？
**A**: 检查权限申请理由是否充分。理由应说明具体使用场景，例如：「用于读取项目群消息，以便追踪项目进度并自动生成日报」。

### Q2: 凭证泄露了怎么办？
**A**: 
1. 立即在飞书开放平台重置凭证
2. 更新 1Password 中的凭证
3. 更新 OpenClaw 配置
4. 审查访问日志，确认是否有异常

### Q3: 如何测试 Bot 权限？
**A**: 使用飞书开放平台的「调试工具」→「权限校验」，输入权限码进行测试。

### Q4: 一个 Bot 可以加入多少个群？
**A**: 企业内部应用没有群数量限制，但建议每个 Bot 只加入相关群组，避免消息路由混乱。
