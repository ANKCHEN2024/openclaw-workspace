#### 自测题目（每题 10 分，满分 100 分）

**Q1**: 解释 Transformer 中的 Self-Attention 机制，它解决了什么问题？
<details>
<summary>参考答案</summary>
Self-Attention 让序列中的每个位置都能关注到其他所有位置，计算权重后加权求和。解决了 RNN 长距离依赖问题，支持并行计算。
</details>

**Q2**: 设计一个 Prompt，让 LLM 生成公司产品介绍文案，要求包含：公司名、核心业务、优势、联系方式。
<details>
<summary>参考答案</summary>

```
你是一名专业文案策划。请为"西安谷风网络科技有限公司"撰写产品介绍文案。
要求：
1. 突出"AI+ 数字孪生"核心业务
2. 强调三维可视化 + 智能化优势
3. 包含数字政府/园区/工厂/水务场景
4. 语气专业但不晦涩
5. 末尾添加联系方式：400-xxx-xxxx
字数：300-500 字
```
</details>

**Q3**: Qwen、Kimi、MiniMax、GLM 四个模型，各适合什么场景？
<details>
<summary>参考答案</summary>
- Qwen：综合能力强，适合复杂任务
- Kimi：长文本处理优秀，适合文档分析
- MiniMax：对话能力强，适合客服场景
- GLM：中文优化好，适合政务场景
</details>

**Q4**: 如何降低 LLM API 调用成本？列举 3 种方法。
<details>
<summary>参考答案</summary>
1. 使用小模型处理简单任务
2. 缓存高频问题答案
3. Prompt 优化减少 token 消耗
4. 本地部署开源模型
</details>

**Q5**: 什么是 Temperature、Top-P、Max Tokens 参数？如何调节？
<details>
<summary>参考答案</summary>
- Temperature：控制随机性（0 确定，1 随机）
- Top-P：核采样阈值（0.9 常用）
- Max Tokens：最大输出长度
- 创意任务调高 Temperature，事实任务调低
</details>

| 上下文管理 | 能处理长上下文场景 | 🟡 |

#### 自测题目

**Q1**: 解释 RAG 的工作流程。
<details>
<summary>参考答案</summary>
1. 文档切分→向量化→存入向量库
2. 用户问题→向量化→检索相似片段
3. 组装上下文 + 问题→LLM 生成答案
4. 返回答案 + 引用来源
</details>

**Q2**: 文本分块有哪些策略？如何选择？
<details>
<summary>参考答案</summary>
1. 固定长度分块（简单，但可能切断语义）
2. 语义分块（按段落/章节，保持完整性）
3. 重叠分块（相邻块有重叠，减少信息丢失）
4. 根据场景选择：技术文档用语义分块，对话用重叠分块
</details>

**Q3**: 如何提升 RAG 检索准确率？
<details>
<summary>参考答案</summary>
1. 优化分块策略
2. 使用更好的 Embedding 模型
3. 混合检索（向量 + 关键词）
4. 重排序（Rerank）
5. 添加元数据过滤
</details>

#### 自测题目

**Q1**: WebSocket 和 HTTP 有什么区别？
<details>
<summary>参考答案</summary>
- HTTP：请求 - 响应模式，单向，短连接
- WebSocket：全双工，双向通信，长连接
- 实时场景用 WebSocket，普通请求用 HTTP
</details>

**Q2**: MQTT 的 QoS 级别有哪些？如何选择？
<details>
<summary>参考答案</summary>
- QoS 0：最多一次（不保证到达）
- QoS 1：至少一次（保证到达，可能重复）
- QoS 2：恰好一次（保证到达且不重复）
- 监控数据用 QoS 1，控制指令用 QoS 2
</details>

**Q3**: 设计一个 IoT 设备接入方案（1000+ 设备）。
<details>
<summary>参考答案</summary>
1. 设备→MQTT Broker（EMQX 集群）
2. Broker→后端（消息队列 Kafka）
3. 后端→数据库（时序数据库 InfluxDB）
4. 后端→前端（WebSocket 推送）
</details>
