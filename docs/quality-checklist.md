## 代码质量标准

### ✅ 必检项（Blocking）

| 检查项 | 标准 | 验证方法 |
|--------|------|----------|
| 语法正确 | 无语法错误 | `bash -n script.sh` |
| 可执行权限 | `.sh` 文件有 `+x` 权限 | `ls -l *.sh` |
| 无硬编码密码 | 不包含明文密码/密钥 | `grep -r "password\s*="` |
| 错误处理 | 关键操作有 `set -e` 或错误捕获 | 代码审查 |
| 路径安全 | 使用 `$WORKSPACE_ROOT` 等变量，非硬编码 | 代码审查 |

### ⚠️ 建议项（Warning）

| 检查项 | 标准 | 验证方法 |
|--------|------|----------|
| 注释完整 | 复杂逻辑有中文注释 | 代码审查 |
| 日志输出 | 关键步骤有 `log_info/error` | 代码审查 |
| 变量命名 | 使用有意义的变量名 | 代码审查 |
| 函数封装 | 重复逻辑提取为函数 | 代码审查 |
| 配置分离 | 可配置项放在脚本开头 | 代码审查 |

### 📝 代码审查要点

```bash
# 1. 运行质量门禁
./scripts/quality-gate.sh strict

# 2. 手动审查清单
- [ ] 脚本头部有用途说明
- [ ] 关键变量有注释
- [ ] 错误处理完整
- [ ] 无调试代码残留
- [ ] 无 commented-out 代码块
```

## 文档质量标准

### ✅ 必检项（Blocking）

| 检查项 | 标准 | 验证方法 |
|--------|------|----------|
| 核心文档存在 | AGENTS.md, SOUL.md, IDENTITY.md, USER.md | `ls -1 *.md` |
| 链接有效 | 内部链接指向存在的文件 | 手动点击/脚本检查 |
| 格式正确 | Markdown 语法正确，无 broken 格式 | 预览检查 |
| 中文书写 | 用户-facing 文档使用中文 | 人工审查 |
| 无敏感信息 | 不包含密码、密钥、私密数据 | `grep` 检查 |

### ⚠️ 建议项（Warning）

| 检查项 | 标准 | 验证方法 |
|--------|------|----------|
| 目录结构 | 有清晰的目录/章节划分 | 人工审查 |
| 更新记录 | 重要文档有版本/更新日志 | 检查文档末尾 |
| 示例代码 | 技术文档包含使用示例 | 人工审查 |
| 跨文档引用 | 相关文档之间有链接 | 人工审查 |
| 每日笔记 | `memory/YYYY-MM-DD.md` 每日创建 | 检查 memory 目录 |

### 📝 文档审查要点

```bash
# 1. 核心文档检查
for doc in AGENTS.md SOUL.md IDENTITY.md USER.md WORK_PRINCIPLES.md TOOLS.md; do
  [ -f "$doc" ] && echo "✓ $doc" || echo "✗ $doc MISSING"
done

# 2. 敏感信息扫描
grep -r "password\|secret\|api_key\|token" --include="*.md" . | grep -v "示例\|example\|<your"

# 3. 链接检查（手动）
# 打开文档，点击所有链接确认有效
```

## 交付物质量标准

### ✅ 任务交付标准

每个 subagent 任务完成前必须检查：

| 检查项 | 标准 | 验证方法 |
|--------|------|----------|
| 交付物完整 | 任务要求的所有文件已创建 | `ls -la` 检查 |
| 功能可用 | 脚本可执行，无报错 | 实际运行测试 |
| 文档更新 | 相关文档已同步更新 | 检查文档变更 |
| 状态记录 | `evolution-state.json` 已更新 | 检查 JSON 变更 |
| 日志记录 | `memory/YYYY-MM-DD.md` 有记录 | 检查记忆文件 |

### 📦 交付物清单模板

```markdown
## 本次交付

### 创建的文件
- [ ] `scripts/xxx.sh` - 功能说明
- [ ] `docs/xxx.md` - 功能说明

### 修改的文件
- [ ] `memory/evolution-state.json` - 能力评分更新
- [ ] `memory/YYYY-MM-DD.md` - 任务记录

### 验证步骤
- [ ] 运行 `./scripts/xxx.sh` 无报错
- [ ] 运行 `./scripts/quality-gate.sh` 通过
- [ ] 相关文档已更新
```

## 验收流程

### 🔄 标准验收流程

```
┌─────────────┐
│  任务完成   │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 自检验收    │ ← subagent 自行检查
│ (本清单)    │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 质量门禁    │ ← 运行 quality-gate.sh
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 主代理审查  │ ← 主代理最终确认
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 交付完成    │
└─────────────┘
```

### 📋 验收检查单

**subagent 自检**（任务完成前）：
- [ ] 所有交付物已创建
- [ ] 脚本可执行且无报错
- [ ] 文档已更新
- [ ] 状态文件已更新
- [ ] 已运行 quality-gate.sh 并通过

**主代理审查**（接收任务前）：
- [ ] 交付物符合任务要求
- [ ] 质量门禁报告无 FAIL 项
- [ ] 能力评分已更新（如适用）
- [ ] 记忆文件有记录

### 🚨 验收不通过处理

如果验收不通过：

1. **记录问题**：在 `memory/YYYY-MM-DD.md` 记录具体问题
2. **返回修复**：将任务返回给 subagent 或创建修复任务
3. **跟踪闭环**：确保问题在 24 小时内修复

## 📈 持续改进

### 质量指标追踪

每周统计以下指标（记录到 `memory/` 或周报）：

| 指标 | 目标 | 计算方法 |
|------|------|----------|
| 首次通过率 | ≥80% | 首次检查即通过的任务数 / 总任务数 |
| 平均修复时间 | <2 小时 | 从发现问题到修复完成的平均时间 |
| 缺陷密度 | <1 个/任务 | 发现的问题数 / 任务数 |
| 文档完整率 | 100% | 核心文档存在数 / 核心文档总数 |

### 清单更新机制

- **每月审查**：每月底审查本清单，更新不适用的项
- **问题驱动**：每次验收发现的问题，考虑是否加入清单
- **版本记录**：在本文档末尾记录重大更新
