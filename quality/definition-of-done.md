# Definition of Done (DoD) - 完成定义

> 核心原则：**未完成定义的工作，不算完成**

---

## 💻 代码任务的 DoD

### 开发阶段

- [ ] **功能实现**
  - [ ] 代码实现完整功能需求
  - [ ] 通过所有单元测试
  - [ ] 通过所有集成测试
  - [ ] 无编译错误/警告

- [ ] **代码质量**
  - [ ] 通过代码审查清单（quality/checklist-code-review.md）
  - [ ] 通过 lint 检查（无错误，警告已评估）
  - [ ] 通过质量检查脚本（scripts/quality-check.sh）
  - [ ] 无敏感信息泄露

- [ ] **性能**
  - [ ] 性能测试通过（如适用）
  - [ ] 无明显性能退化
  - [ ] 资源使用合理（内存、CPU、网络）

### 测试阶段

- [ ] **单元测试**
  - [ ] 新增代码测试覆盖率 ≥ 80%
  - [ ] 所有测试通过（绿色）
  - [ ] 测试包含边界条件
  - [ ] 测试包含错误处理

- [ ] **集成测试**
  - [ ] API 测试通过
  - [ ] 端到端测试通过（关键路径）
  - [ ] 回归测试通过

- [ ] **手动测试**
  - [ ] 功能在开发环境验证
  - [ ] 功能在测试环境验证
  - [ ] 用户体验符合预期

### 文档阶段

- [ ] **代码文档**
  - [ ] 公共 API 有 JSDoc/文档注释
  - [ ] 复杂逻辑有解释注释
  - [ ] 类型定义完整

- [ ] **用户文档**
  - [ ] README 已更新（如有新功能）
  - [ ] 使用示例已添加
  - [ ] 配置说明已更新

- [ ] **变更文档**
  - [ ] CHANGELOG.md 已更新
  - [ ] 破坏性变更已标注
  - [ ] 迁移指南已编写（如需要）

### 部署阶段

- [ ] **代码审查**
  - [ ] PR/MR 已创建
  - [ ] 至少 1 人审查通过
  - [ ] 所有审查意见已处理

- [ ] **CI/CD**
  - [ ] CI 流水线通过
  - [ ] 构建成功
  - [ ] 部署脚本已更新（如需要）

- [ ] **发布**
  - [ ] 版本号已更新（语义化版本）
  - [ ] Git tag 已创建
  - [ ] 代码已合并到主分支

## 📝 内容任务的 DoD

### 调研阶段

- [ ] **信息收集**
  - [ ] 主题相关核心资料已收集
  - [ ] 数据来源可靠且已标注
  - [ ] 竞品信息已分析（如适用）
  - [ ] 目标受众需求已明确

- [ ] **大纲设计**
  - [ ] 内容结构已规划
  - [ ] 关键要点已列出
  - [ ] 逻辑流程已设计
  - [ ] 预期长度已确定

### 撰写阶段

- [ ] **初稿完成**
  - [ ] 所有内容要点已覆盖
  - [ ] 逻辑结构清晰
  - [ ] 语言流畅易懂
  - [ ] 配图/图表已添加（如需要）

- [ ] **质量检查**
  - [ ] 通过内容审查清单（quality/checklist-content-review.md）
  - [ ] 事实/数据已核实
  - [ ] 链接已测试（无 404）
  - [ ] 拼写/语法已检查

### 审核阶段

- [ ] **自我审核**
  - [ ] 放置 24 小时后重新审阅
  - [ ] 以读者视角重新评估
  - [ ] 修改不满意的部分

- [ ] **同行评审**
  - [ ] 至少 1 人审阅并提供反馈
  - [ ] 所有反馈已评估
  - [ ] 合理建议已采纳

- [ ] **最终检查**
  - [ ] 格式规范统一
  - [ ] 标题/目录正确
  - [ ] 引用/来源完整
  - [ ] SEO 优化（如适用）

### 发布阶段

- [ ] **发布准备**
  - [ ] 发布平台已确定
  - [ ] 发布时间已规划
  - [ ] 推广计划已制定（如需要）

- [ ] **发布执行**
  - [ ] 内容已发布
  - [ ] 社交媒体已同步（如需要）
  - [ ] 相关链接已更新

- [ ] **发布后**
  - [ ] 发布链接已记录
  - [ ] 读者反馈渠道已开放
  - [ ] 数据追踪已设置（如适用）

## 🤖 Subagent 任务的 DoD

### 目标定义

- [ ] **任务清晰**
  - [ ] 任务目标已明确定义
  - [ ] 成功标准已量化
  - [ ] 交付物已具体描述
  - [ ] 时间预算已确定

- [ ] **边界明确**
  - [ ] 任务范围已界定
  - [ ] 不包含的内容已说明
  - [ ] 依赖关系已识别
  - [ ] 风险已评估

### 执行过程

- [ ] **自主决策**
  - [ ] 已理解任务背景
  - [ ] 已制定执行计划
  - [ ] 关键决策已记录
  - [ ] 遇到问题已自主解决

- [ ] **进度跟踪**
  - [ ] 任务分解合理
  - [ ] 里程碑已达成
  - [ ] 时间使用高效
  - [ ] 无等待/阻塞

### 交付物

- [ ] **成果完整**
  - [ ] 所有要求的交付物已完成
  - [ ] 交付物质量符合标准
  - [ ] 交付物已保存到正确位置
  - [ ] 交付物命名规范

- [ ] **文档齐全**
  - [ ] 执行过程已记录
  - [ ] 关键决策已说明
  - [ ] 使用指南已提供（如需要）
  - [ ] 后续建议已给出

### 验证标准

- [ ] **功能验证**
  - [ ] 交付物可正常使用
  - [ ] 满足任务目标
  - [ ] 通过质量检查
  - [ ] 无已知缺陷

- [ ] **集成验证**
  - [ ] 与现有系统兼容
  - [ ] 不影响现有功能
  - [ ] 已通知相关人员（如需要）
  - [ ] 已更新相关文档

### 汇报要求

- [ ] **结果汇报**
  - [ ] 完成情况已总结
  - [ ] 关键成果已列出
  - [ ] 遇到的问题已说明
  - [ ] 后续建议已提供

- [ ] **状态标记**
  - [ ] 任务状态已更新
  - [ ] 相关文件已标记
  - [ ] 记忆已更新（如需要）
  - [ ] 主 agent 已通知

## 🎯 通用验收标准

### 质量门槛

| 任务类型 | 最低要求 | 目标标准 |
|---------|---------|---------|
| 代码任务 | 所有安全项通过 | 0 失败，≤3 警告 |
| 内容任务 | 所有准确性项通过 | B 级以上评分 |
| Subagent 任务 | 交付物完整可用 | 超出预期 |

### 时间管理

- **小任务** (<1 小时): 直接完成，无需中期汇报
- **中任务** (1-4 小时): 每 2 小时简要更新进度
- **大任务** (>4 小时): 分解为子任务，分别验收

### 沟通原则

- **自主决策**: 在任务范围内自主决定，无需等待批准
- **透明汇报**: 完成后立即汇报结果，包括问题和教训
- **主动升级**: 遇到阻塞 >30 分钟，主动寻求帮助
