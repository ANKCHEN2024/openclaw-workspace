# 代码质量检查清单

> 使用前说明：在代码提交前逐项检查，所有项目必须为"是"才能合并

## 📋 代码规范

- [ ] **命名规范**
  - [ ] 变量名是否清晰表达用途（避免 single-letter 命名）
  - [ ] 函数名是否使用动词开头（如 `getUserInfo`, `calculateTotal`）
  - [ ] 常量名是否使用全大写 + 下划线（如 `MAX_RETRY_COUNT`）
  - [ ] 类名是否使用 PascalCase
  - [ ] 私有成员是否使用约定前缀（如 `_privateMethod`）

- [ ] **注释质量**
  - [ ] 复杂算法是否有注释解释思路
  - [ ] 公共 API 是否有 JSDoc/文档注释
  - [ ] 注释是否解释"为什么"而非"是什么"
  - [ ] 是否没有过时的注释（代码已改但注释未更新）
  - [ ] 是否没有废话注释（如 `i++ // i 加 1`）

- [ ] **代码结构**
  - [ ] 单个函数是否不超过 50 行
  - [ ] 单个文件是否不超过 500 行
  - [ ] 是否有合理的模块/文件拆分
  - [ ] 是否避免深层嵌套（不超过 3 层）
  - [ ] 是否遵循单一职责原则

## ✅ 功能完整性

- [ ] **单元测试**
  - [ ] 新增功能是否有对应的单元测试
  - [ ] 测试覆盖率是否达到项目要求（>80%）
  - [ ] 测试是否覆盖正常路径
  - [ ] 测试是否覆盖边界条件（空值、极大值、极小值）
  - [ ] 测试是否覆盖错误路径（异常处理）

- [ ] **边界条件**
  - [ ] 是否处理了 null/undefined 输入
  - [ ] 是否处理了空数组/空字符串
  - [ ] 是否处理了网络超时/失败
  - [ ] 是否处理了并发/竞态条件
  - [ ] 是否处理了数据格式异常

- [ ] **集成测试**
  - [ ] 关键路径是否有端到端测试
  - [ ] API 接口是否有集成测试
  - [ ] 数据库操作是否有事务测试

## ⚡ 性能考虑

- [ ] **时间复杂度**
  - [ ] 是否避免在循环中执行 O(n) 操作
  - [ ] 是否使用合适的数据结构（Map vs Object）
  - [ ] 是否避免不必要的嵌套循环
  - [ ] 是否有明显的性能瓶颈需要优化

- [ ] **缓存策略**
  - [ ] 重复计算是否使用缓存
  - [ ] API 请求是否有适当的缓存
  - [ ] 数据库查询是否有索引优化
  - [ ] 静态资源是否有 CDN 缓存

- [ ] **内存管理**
  - [ ] 是否有内存泄漏风险（未清理的定时器、监听器）
  - [ ] 大对象是否及时释放
  - [ ] 是否避免在循环中创建对象

## 🔒 安全审查

- [ ] **敏感数据**
  - [ ] 代码中是否没有硬编码的密码/密钥
  - [ ] 是否没有提交 .env 文件
  - [ ] Token 是否从环境变量读取
  - [ ] 敏感信息是否加密存储

- [ ] **注入风险**
  - [ ] SQL 查询是否使用参数化（防 SQL 注入）
  - [ ] 用户输入是否进行转义（防 XSS）
  - [ ] 文件路径是否验证（防路径遍历）
  - [ ] 命令执行是否过滤输入（防命令注入）

- [ ] **权限控制**
  - [ ] API 是否有身份验证
  - [ ] 敏感操作是否有权限检查
  - [ ] 是否实现最小权限原则

## 📚 文档完整性

- [ ] **README**
  - [ ] 是否有项目简介
  - [ ] 是否有安装/运行说明
  - [ ] 是否有使用示例
  - [ ] 是否有贡献指南

- [ ] **代码注释**
  - [ ] 公共函数是否有文档注释
  - [ ] 复杂逻辑是否有解释注释
  - [ ] 类型定义是否有说明

- [ ] **变更日志**
  - [ ] 是否有 CHANGELOG.md 或更新记录
  - [ ] 变更是否记录在案
  - [ ] 破坏性变更是否有明确标注

---

## 🎯 快速检查命令

```bash
# 运行质量检查脚本
./scripts/quality-check.sh

# 运行测试
npm test

# 检查代码风格
npm run lint

# 检查安全漏洞
npm audit
```

## 📊 评分标准

- **优秀 (A)**: 所有检查项通过，无警告
- **良好 (B)**: 所有关键项通过，≤3 个非关键项待改进
- **合格 (C)**: 所有安全项通过，≤5 个待改进项
- **不合格 (F)**: 任何安全项未通过，或 >5 个待改进项

**合并要求**: 至少达到 C 级，安全项必须 100% 通过
