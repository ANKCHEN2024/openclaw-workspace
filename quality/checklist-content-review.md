# 内容质量检查清单

> 使用前说明：在内容发布前逐项检查，所有项目必须为"是"才能发布

## 🎯 准确性

- [ ] **事实核查**
  - [ ] 所有数据是否有可靠来源
  - [ ] 引用的研究/报告是否注明出处
  - [ ] 统计数据是否为最新版本
  - [ ] 是否没有未经证实的断言
  - [ ] 技术术语使用是否准确

- [ ] **数据验证**
  - [ ] 数字计算是否正确
  - [ ] 百分比/比例是否合理
  - [ ] 时间线/日期是否准确
  - [ ] 人名/地名/公司名拼写是否正确
  - [ ] 链接是否有效且指向正确内容

- [ ] **信息时效性**
  - [ ] 内容是否反映最新情况
  - [ ] 过时的信息是否已更新或标注
  - [ ] 是否有"最后更新时间"标注

## 📦 完整性

- [ ] **要点覆盖**
  - [ ] 是否覆盖主题的所有关键方面
  - [ ] 是否回答了读者可能的主要问题
  - [ ] 是否有必要的背景信息
  - [ ] 是否有清晰的结论/总结
  - [ ] 是否有下一步行动建议（如适用）

- [ ] **结构完整**
  - [ ] 是否有清晰的引言
  - [ ] 是否有逻辑的主体部分
  - [ ] 是否有有力的结论
  - [ ] 各部分之间是否有过渡
  - [ ] 是否有目录/导航（长内容）

- [ ] **补充材料**
  - [ ] 是否需要配图/图表（已添加）
  - [ ] 是否需要代码示例（已添加）
  - [ ] 是否需要参考资源（已添加）
  - [ ] 是否需要相关推荐阅读（已添加）

## 💡 清晰度

- [ ] **逻辑结构**
  - [ ] 内容是否有清晰的层次结构
  - [ ] 段落之间是否逻辑连贯
  - [ ] 论点是否有论据支撑
  - [ ] 是否有清晰的因果关系
  - [ ] 是否避免跳跃式思维

- [ ] **表达简洁**
  - [ ] 是否没有冗余重复的内容
  - [ ] 句子是否简洁明了（平均<25 字）
  - [ ] 是否避免过度使用被动语态
  - [ ] 是否避免行话/术语堆砌
  - [ ] 复杂概念是否有通俗解释

- [ ] **可读性**
  - [ ] 是否使用小标题分段
  - [ ] 是否使用列表/表格呈现信息
  - [ ] 关键信息是否有强调（加粗/高亮）
  - [ ] 段落长度是否适中（3-5 句）
  - [ ] 是否有适当的留白

## 💎 价值度

- [ ] **可操作性**
  - [ ] 读者是否能根据内容采取行动
  - [ ] 是否有具体的步骤/方法
  - [ ] 是否有实用的工具/资源推荐
  - [ ] 是否有可复用的模板/框架
  - [ ] 是否有清晰的预期结果

- [ ] **独特见解**
  - [ ] 是否有原创观点/分析
  - [ ] 是否提供新的视角/方法
  - [ ] 是否有个人经验/案例分享
  - [ ] 是否超越表面信息提供深度
  - [ ] 是否有数据/研究支撑观点

- [ ] **读者价值**
  - [ ] 内容是否解决读者实际问题
  - [ ] 读者是否能获得新知识/技能
  - [ ] 内容是否节省读者时间
  - [ ] 读者是否有"啊哈"时刻
  - [ ] 读者是否会收藏/分享

## 📐 格式规范

- [ ] **Markdown 格式**
  - [ ] 标题层级是否正确（H1→H2→H3）
  - [ ] 列表格式是否统一
  - [ ] 代码块是否指定语言
  - [ ] 引用是否有正确格式
  - [ ] 表格是否对齐整齐

- [ ] **链接规范**
  - [ ] 链接是否有描述性文字
  - [ ] 外部链接是否在新窗口打开
  - [ ] 链接是否经过测试（无 404）
  - [ ] 是否有必要的来源标注
  - [ ]  affiliate 链接是否披露

- [ ] **引用规范**
  - [ ] 引用是否有明确标注
  - [ ] 图片是否有来源/版权说明
  - [ ] 数据是否有出处
  - [ ] 是否有参考文献列表
  - [ ] 是否遵守版权规定

- [ ] **视觉呈现**
  - [ ] 图片是否清晰且相关
  - [ ] 图片是否有 alt 文本
  - [ ] 是否有适当的图表/可视化
  - [ ] 排版是否美观一致
  - [ ] 是否有品牌元素（如适用）

---

## 🎯 快速检查命令

```bash
# 运行内容质量检查
./scripts/quality-check.sh --content <file.md>

# 检查链接有效性
npx markdown-link-check <file.md>

# 检查拼写
npx cspell <file.md>
```

## 📊 评分标准

- **优秀 (A)**: 所有检查项通过，内容具有独特价值
- **良好 (B)**: 所有关键项通过，≤3 个非关键项待改进
- **合格 (C)**: 所有准确性项通过，≤5 个待改进项
- **不合格 (F)**: 任何准确性项未通过，或 >5 个待改进项

**发布要求**: 至少达到 B 级，准确性项必须 100% 通过

## 🔍 同行评审检查

对于重要内容（教程、文档、文章），建议进行同行评审：

- [ ] 是否请同事/朋友阅读并提供反馈
- [ ] 是否收集了目标读者的意见
- [ ] 是否根据反馈进行了修改
- [ ] 是否有第二人校对错别字/语法
