# AI 短剧平台 2.0 - 产品需求文档

## Overview
- **Summary**: AI 短剧平台 2.0 是对现有项目的全面升级，通过整合现有分散的模块、完善后端服务、优化用户体验，打造一个真正可投入生产使用的全流程 AI 短剧生成系统。
- **Purpose**: 解决当前项目中存在的模块分散、缺少后端整合、无数据库实现、无真正可运行系统等问题，提供从小说到成片的完整自动化工作流。
- **Target Users**: 内容创作者、短视频运营者、新媒体工作室、个人自媒体

## Goals
- 整合现有所有分散模块为完整系统
- 实现完整的后端服务与数据库
- 提供真正可运行的全流程 AI 短剧生成
- 优化前端交互体验，支持可视化编辑
- 增强视频质量与内容一致性
- 支持团队协作与项目版本管理

## Non-Goals (Out of Scope)
- 不开发新的 AI 模型（继续使用现有国产 API）
- 不实现移动端 App（专注 Web 端）
- 不开发支付系统（2.0 版本暂不涉及商业化）
- 不实现复杂的推荐算法系统
- 不做大规模分布式架构设计（保留可扩展性）

## Background & Context
当前项目状态分析：
- **优点**：已有完整的架构设计文档，核心算法模块（故事分析、人物构建、场景构建、视频生成等）都有初步实现
- **问题**：
  1. 模块完全分散，没有后端整合
  2. 数据库设计文档存在，但没有实际代码实现
  3. 前端框架存在，但与后端无连接
  4. 缺少任务调度、队列管理、文件存储等基础设施
  5. 无完整的端到端工作流
  6. 缺少错误处理、监控、日志等生产级特性

## Functional Requirements
- **FR-1**: 完整的用户认证与权限管理系统
- **FR-2**: 项目全流程管理（创建 → 导入小说 → 分析 → 编辑 → 生成 → 导出）
- **FR-3**: 小说智能分析与自动分集
- **FR-4**: 人物/场景可视化编辑与一致性管理
- **FR-5**: 分镜编辑与预览
- **FR-6**: 视频生成任务调度与进度监控
- **FR-7**: 音频合成与字幕自动生成
- **FR-8**: 多轨道视频合成与导出
- **FR-9**: 素材库管理（上传、分类、搜索）
- **FR-10**: 项目版本管理与协作

## Non-Functional Requirements
- **NFR-1**: 单集（60秒）生成时间 ≤ 20 分钟
- **NFR-2**: 支持同时 5 个并发任务
- **NFR-3**: API 响应时间 ≤ 2秒（非生成类请求）
- **NFR-4**: 系统可用性 ≥ 99%
- **NFR-5**: 人物/场景一致性 ≥ 90%
- **NFR-6**: 前端页面加载时间 ≤ 3秒

## Constraints
- **Technical**: 必须使用 100% 国产 AI API，使用 Node.js + PostgreSQL + Redis 技术栈
- **Business**: 2.0 版本需在合理时间内完成开发，优先保证核心功能可用
- **Dependencies**: 依赖现有模块的算法实现，依赖通义千问、可灵 AI、即梦 AI、阿里云语音等国产服务

## Assumptions
- 用户已拥有各 AI 服务的 API Key
- 服务器配置满足最低要求（4核 8G 以上）
- FFmpeg 已正确安装在服务器上
- 用户具备基本的计算机操作能力

## Acceptance Criteria

### AC-1: 用户可以注册登录并管理账户
- **Given**: 系统已部署运行
- **When**: 用户访问系统注册/登录页面，输入有效信息
- **Then**: 用户成功创建账户或登录，可查看和编辑个人资料
- **Verification**: `programmatic`

### AC-2: 用户可以创建项目并导入小说
- **Given**: 用户已登录
- **When**: 用户点击创建项目，输入项目信息并粘贴小说文本
- **Then**: 项目创建成功，小说保存到数据库
- **Verification**: `programmatic`

### AC-3: 系统可以自动分析小说并生成分集
- **Given**: 项目已创建且包含小说
- **When**: 用户点击"开始分析"
- **Then**: 系统自动分析小说，提取人物、场景，生成分集列表
- **Verification**: `programmatic` + `human-judgment`

### AC-4: 用户可以可视化编辑人物和场景
- **Given**: 小说已分析完成
- **When**: 用户进入人物/场景编辑页面，修改信息并重新生成图像
- **Then**: 更改保存成功，图像重新生成，保持一致性
- **Verification**: `programmatic` + `human-judgment`

### AC-5: 系统可以生成并编辑分镜
- **Given**: 分集已创建
- **When**: 用户进入分镜编辑页面，可查看和编辑每个镜头
- **Then**: 分镜生成并展示，支持编辑修改
- **Verification**: `programmatic` + `human-judgment`

### AC-6: 用户可以发起视频生成任务并查看进度
- **Given**: 分镜已完成
- **When**: 用户点击"生成视频"
- **Then**: 任务进入队列，用户可实时查看进度，最终获得完整视频
- **Verification**: `programmatic`

### AC-7: 视频包含配音、字幕和背景音乐
- **Given**: 视频生成任务已完成
- **When**: 用户预览生成的视频
- **Then**: 视频包含角色配音、同步字幕和合适的背景音乐
- **Verification**: `human-judgment`

### AC-8: 用户可以管理素材库
- **Given**: 用户已登录
- **When**: 用户上传素材或搜索现有素材
- **Then**: 素材成功上传/找到并可用于项目
- **Verification**: `programmatic`

### AC-9: 系统稳定运行，异常有友好提示
- **Given**: 系统在使用中
- **When**: 发生网络错误、API 失败等异常情况
- **Then**: 系统显示友好的错误提示，不崩溃，可重试
- **Verification**: `programmatic` + `human-judgment`
